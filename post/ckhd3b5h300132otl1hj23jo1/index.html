重学前端笔记（6）

前端 JavaScript 闭包 执行上下文
学习笔记
发布日期:   2020-11-05

闭包

闭包其实只是一个绑定了执行环境的函数

所以说闭包本质上是一个函数，特殊之处在于他绑定了一个执行环境。

闭包的组成部分

  • 环境部分
    • 环境：函数的词法环境（执行上下文的一部分）
    • 标识符列表：函数中用到的未声明的变量
  • 表达式部分：函数体

其实早期我一直把执行上下文当成闭包，实际上 JS 中和闭包对应的概念是函数，不是执行上下文或者作用域。

另外 MDN 对于闭包的解释是：闭包是指那些能够访问自由变量的函数，这里也验证了闭包本质上来说就是一个函数。

但是什么是自由变量呢？自由变量是指在函数中使用的，但既不是函数参数也不是函数的局部变量的变量。

var a = 1

function foo() {
    console.log(a)
}

foo()

这里的 a 就是一个自由变量。

综合一下我们可以说闭包 = 函数 + 函数能够访问的自由变量。

执行上下文：执行的基础设施

JS 中，词法环境（闭包中的环境部分）只是执行上下文的一部分，不是全部。

JS 标准中把一段代码（包括函数），执行所需要的所有信息定义为“执行上下文”。

说了这么半天其实都是从标准上来说闭包、执行上下文是个啥东西，但是还是要放到代码上来看。

代码推导什么是执行上下文

观察以下代码

var b = {}
let c = 1
this.a = 2

思考几个问题：

  1. var 把 b 声明到哪里；
  2. b 表示哪个变量；
  3. b 的原型是哪个对象；
  4. let 把 c 声明到哪里；
  5. this 指向哪个对象。

var 声明与赋值

我们首先要明确一点，var 声明作用域是函数执行的作用域，也就是说它会穿透 for、if 等语句。

既然能穿透，就有技巧能够控制 var 在特定范围内生效，老版本的 JS 中有一个叫做立即执行的函数表达式（IIFE）。通过创建一个函数，并且立即执行，来构造一个新的域。

;(function() {
    var a = 4
})()

之前大家都写分号，放到现在不写分号就有一个问题，如果 IIFE 上面有一个函数，没写分号，那么 IIFE 的括号会被当成上一个函数的调用，所以有些规范要求写分号。

但这样很怪，所以推荐使用 void 关键字。

void function() {
    var a = 4
    console.log(a)
}()

let

let 会产生一个块级作用域，比起 var 来说好很多。

函数

在 JS 中切换上下文最常见的场景就是在函数调用中，首先我们先给函数分个类，然后一个一个看看。

  1. 普通函数：function 关键字定义的
  2. 箭头函数：() => {}
  3. 在 calss 里面定义的函数：也就是我们做封装时候的方法
  4. 生成器函数：function * 定义的
  5. 用 class 定义的类：类的本质还是函数
  6. 异步函数：添加 async 关键字的函数

this

this 是执行上下文中很重要的一个组成部分，同一个函数调用的方式不同，得到的 this 值也不一样。

function showThis() { console.log(this) }

var o = { showThis: showThis }

showThis()  // global
o.showThis()  // o

普通函数的 this 值由“调用它所使用的引用”决定的，简单说就是谁调用，this 就指向谁。

const showThis = () => { console.log(this) }

var o = { showThis: showThis }

showThis()  // global
o.showThis()  // global

改成箭头函数之后，无论用什么引用调用它，都不影响他的 this 值。
