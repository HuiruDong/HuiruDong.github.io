重学前端笔记（7）

前端 HTML 标签
学习笔记
发布日期:   2020-11-05

到 HTML 部分了！

head 标签

  1. 必须包含一个 <title> ，并且最多只能包含一个 <base>
  2. 如果文档作为 iframe，或者其它方式指定了文档标题，可以不包含 <title>

base 标签

这是个历史遗留问题，它的作用是给页面上所有的 URL 提供一个基础，这就导致它会改变全局的链接地址，很容易影响 JS。

meta 标签

这个孩子包含的东西有点多。
首先明确的是 <meta> 是一组键值对，用来写元信息的。可以有多个，一般来说每个标签都有 name 和 content 两个属性来定义的。

<meta name="viewport" content="width=device-width, initial-scale=1.0">

当然这是常规操作，随着写代码的人越来越多，总有那么几种常见场景大家都用，所以简写也就出现了。

有 charset 的

<meta charset="UTF-8">

有 http-equiv 的

<meta http-equiv="content-type" content="text/html; charset=UTF-8">

这种 <meta> 表示执行一个命令，这种情况就不需要 name 了。

http-equiv 的值可以是：

  • content-type：指定内容类型
  • content-language：指定内容语言
  • default-style：指定默认样式表
  • refresh：刷新
  • set-cookie
  • x-ua-compatible
  • content-security-policy

name=”viewport”

<!-- 移动端标准 meta -->
<meta name="viewport" content="width=device-width,initial-scale=1,minimum-scale=1,maximum-scale=1,user-scalable=no">

移动端常见哈~

  • whidth：页面宽。数值 or device-width
  • height：页面高。数值 or device-height
  • initial-scale：初始缩放比例
  • minimum-scale：最小缩放比例
  • maximum-scale：最大缩放比例
  • user-scalable：是否允许用户缩放

其他标准中有的

  • author: 页面作者
  • description：页面描述，这个属性可能被用于搜索引擎或者其它场合
  • generator: 生成页面所使用的工具，主要用于可视化编辑器，如果是手写 HTML 的网页，不需要加这个 meta
  • keywords: 页面关键字，对于 SEO 场景非常关键
  • referrer: 跳转策略，是一种安全考量
  • theme-color: 页面风格颜色，实际并不会影响页面，但是浏览器可能据此调整页面之外的 UI（如窗口边框或者 tab 的颜色）
