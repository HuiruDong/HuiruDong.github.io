圣杯布局

<style>
    /* 负边距的体现在于修改了元素的尺寸，这个尺寸是说在计算的时候的尺寸，但是肉眼看上去的尺寸是没有变的 */
    * {
        margin: 0;
        padding: 0;
    }

    .container {
        overflow: hidden;
        padding: 0 100px;
    }

    .inner {
        position: relative;
        width: 100%;
        float: left;
        height: 80px;
        background: green;
    }

    .left {
        position: relative;
        width: 100px;
        float: left;
        /* 因为 container 是有 100 的边距的，所以想要顶死在浏览器边上，需要 -100px */
        left: -100px;
        height: 80px;
        /* 设置边上的时候会被挤下去，设置 margin-left 会被提上来 */
        margin-left: -100%;
        background: yellow;
    }

    .right {
        /* 当左边设置结束后，右边的盒子占据了做左边的位置，也就是 padding 100px 的位置 */
        position: relative;
        width: 100px;
        float: left;
        right: -100px;  
        height: 80px;
        /* 上移靠右 */
        margin-left: -100px;
        background: pink
    }
</style>

<div class="container">
    <div class="inner">开始我是中间盒子我是中间盒子我是中间盒子我是中间盒子我是中间盒子我是中间盒子我是中间盒子我是中间盒子我是中间盒子</div>
    <div class="left">开始我是左边盒子我是左边盒子我是左边盒子我是左边盒子我是左边盒子我是左边盒子</div>
    <div class="right">开始我是右边盒子我是右边盒子我是右边盒子我是右边盒子我是右边盒子我是右边盒子</div>
</div>

这里需要注意几点：

  1. 给容器设置左右 padding
  2. float 的顺序，先是 inner，要先把 inner 放好。然后是 left 和 right
  3. left 和 right 需要设置 position: relative 让它们能动起来
  4. 利用负边距让 left 和 right 哪凉快哪待着

对于圣杯布局和双飞翼布局，都在解决一个问题，就是两栏固定，中间自适应。他们的区别在于解决 inner 盒子里内容不被覆盖的问题的方案不同。

在让三个盒子形成三栏布局的方案圣杯和双飞翼是一样的，三者浮动，左右利用 -margin 让三者排排坐。

接下来解决 inner 盒子内容不被遮挡问题，圣杯采取了用 padding 挤，左右两栏用 position: relative 保证不会挡着中间。而双飞翼是新建一个 div，用 margin 保证自己的位置不被遮挡。

简单说就是：双飞翼布局比圣杯布局多创建了一个div，但不用相对布局了

我个人感觉对于双飞翼和圣杯来说，用 flex 解决可能更方便。

<style>
    * {
        padding: 0;
        margin: 0;
    }
    .container {
        width: 340px;
        border: 1px solid lightseagreen;
        margin: 30px auto;
        display: flex;
        padding: 10px;
    }

    .container div {
        height: 300px;
    }

    .left {
        background: lightseagreen;
    }

    .right {
        background: lightgrey;
        /* 这里的缩放比例 */
        flex: 1;    
    }

</style>

<div class="container">
    <div class="left" style="width: 100px;"></div>
    <div class="right"></div>
    <div class="left" style="width: 100px;"></div>
</div>

多列等高

<style>
    .container {
        width: 300px;
        margin: 40px auto;
        border: 2px solid lightseagreen;
        overflow: hidden;
    }

    .container .nav {
        border-right: 2px solid lightseagreen;
        float: left;
        width: 100px;
        /* 这里是重点 */
        padding-bottom: 1000px;
        margin-bottom: -1000px;
    }

    .container .content {
        float: left;
        width: 198px;
        /* 这里是重点 */
        padding-bottom: 1000px;
        margin-bottom: -1000px;
    }
</style>

<div class="container">
    <div class="nav">
        <p>nav action</p>
        <p>nav action</p>
        <p>nav action</p>
        <p>nav action</p>
    </div>
    <div class="content">
        <P>content</P>
        <P>content</P>
        <P>content</P>
        <P>content</P>
        <P>content</P>
        <P>content</P>
        <P>content</P>
        <P>content</P>
        <P>content</P>
        <P>content</P>
    </div>
</div>

原理在于给“列”设置一个足够大的 padding-bottom ，同时设置一个相同的负边距 margin-bottom ，这样两两相抵，父元素的高度就变成了原本没有设置之前的高度，如果有任意一列的高度增加了，父元素就会乘起来，短的那一列就会用 padding 补起来。

上面的版本是早期社区通过各种黑科技来解决等高问题，后面提出的 flex 布局其实为这类“盒子根据外部尺寸变化而变化”的问题提供了很好的解决方案。我们来看一下 flex 布局的方案。

<style>
    * {
        padding: 0;
        margin: 0;
    }
    .container {
        width: 300px;
        border: 1px solid lightseagreen;
        margin: 30px auto;
        display: flex;
        align-items: auto;
        align-content: center;
    }

    .container div {
        width: 100px;
        margin: 10px;
    }

    .left {
        margin-right: 5px;
        background: lightskyblue;
    }

    .right {
        margin-left: 5px;
        background: lightsalmon;
    }

</style>

<div class="container">
    <div class="left" style="height: 300px;"></div>
    <div class="right"></div>
</div>

其实 flex 还是挺强大的。
